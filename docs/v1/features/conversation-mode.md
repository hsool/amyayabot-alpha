# 1:1 대화 모드 완벽 가이드

에이미가 당신의 이름을 부르면 자동으로 시작되는 대화 기능입니다. 여러 번 주고받으며 대화하고, 중간에 투표나 추첨 같은 게임을 시작할 수도 있습니다.

## 대화 모드가 시작되는 원리

### 호명 감지 (Name Call Detection)

**쉽게 말하면:** 당신이 캐릭터의 이름을 부르면, 에이미가 귀를 쫑긋 세우고 대화를 시작합니다.

**정확히 어떻게 작동하나요?**

1. 에이미가 계속 당신의 음성을 듣고 있습니다 (STT - Speech To Text)
2. 매 0.5초마다 최신 음성이 텍스트로 변환됩니다
3. 에이미는 그 텍스트에서 자신의 이름을 찾습니다
4. 이름을 발견하면 → **대화 모드 시작!**

### 인식되는 이름들

에이미는 다음 형태의 호명을 모두 인식합니다:

- **기본 이름**: "에이미" (설정에서 캐릭터 이름으로 등록된 이름)
- **호명형 자동 확장**: "에이미" → "에이미야", "에이미아" 자동 인식
- **추가 트리거**: 설정에서 추가로 등록한 별칭들

**예시**
```
당신: "에이미!"           → 감지 O
당신: "에이미야!"         → 감지 O (자동 확장)
당신: "에이미아"          → 감지 O (자동 확장)
당신: "에이미, 뭐해?"     → 감지 O
당신: "지금 뭐해?"        → 감지 X (이름 없음)
```

### 오감지 방지

호명을 감지한 후, 에이미는 다시 호명되기까지 **최소 대기 시간**을 둡니다. 기본값은 3초입니다. 이는 TV나 음악에서 우연히 유사한 소리가 감지되는 것을 방지합니다.

또한 **TTS가 재생 중이면 호명 감지를 하지 않습니다**. 자신의 음성으로 인한 오감지를 막기 위해서입니다.

## 대화의 흐름

### 단계별 과정

```
1. 호명 감지
   ↓
2. 약 0.5~1초 대기 (자연스러운 느낌을 위해)
   ↓
3. AI 응답 생성 (최대 1번의 턴)
   ↓
4. 말풍선과 TTS로 표현
   ↓
5. 청취 대기 (다음 발화 기다림)
   ↓
6. 조건 만족 시 종료 (타임아웃 또는 최대 턴 도달)
```

### 대화 턴 (Turn)

**턴이란?** 대화에서 한 사람이 말하는 한 번의 차례입니다.

- **1턴**: "에이미!" (당신) → 에이미의 첫 응답
- **2턴**: 당신의 두 번째 발화 → 에이미의 두 번째 응답
- **...계속**

대화 모드에는 **최대 턴 제한**이 있습니다. 기본값은 10턴입니다.

```
예시 대화:
당신: "에이미야" (1턴 시작)
에이미: "네! 뭘 도와드릴까요?" (1턴 응답)

당신: "오늘 날씨 어때?" (2턴)
에이미: "오늘 날씨가 정말 좋네요!" (2턴 응답)

...계속 반복...

에이미: "마지막 질문이에요!" (10턴 응답 - 최대 도달)
→ 대화 모드 자동 종료
```

## 대화 종료 조건

대화 모드는 다음 중 하나의 조건에서 자동으로 종료됩니다:

### 1. 최대 턴 도달

- **10턴 완료** → 자동 종료
- 설정에서 변경 가능 (1~50턴)

### 2. 타임아웃 (침묵 감지)

**기본 타임아웃: 30초**

당신이 30초 이상 말하지 않으면 대화가 끝납니다.

하지만 에이미는 **똑똑한 타임아웃**을 씁니다:

| 상황 | 타임아웃 |
|------|---------|
| 에이미가 질문을 물었을 때<br>(예: "뭐가 하고 싶어?") | 60초 |
| 대화가 활발할 때<br>(1턴에 8초 미만) | 45초 |
| 대화가 느릴 때<br>(1턴에 20초 초과) | 15초 |
| 보통 속도 | 30초 |

이렇게 동적으로 조정함으로써 자연스러운 대화 흐름을 유지합니다.

### 3. 장시간 스킬 실행

투표나 추첨처럼 오래 걸리는 기능이 실행되면, 대화 모드가 **먼저 종료**됩니다. 그 기능에 집중하기 위해서입니다.

## 대화 중 스킬 사용

### "스킬"이란?

스킬은 투표, 추첨, 룰렛처럼 상호작용이 있는 특별한 기능들입니다.

### 대화 중 스킬을 어떻게 트리거할까?

**자연스럽게 말하기** - 에이미가 알아서 인식합니다!

```
당신: "에이미야, 투표 시작해볼까?"
에이미: "좋아! 투표 항목이 뭔데?" → 투표 스킬 시작

당신: "에이미, 추첨을 한 번 해보자!"
에이미: "오케이! 몇 명을 뽑을까?" → 추첨 스킬 시작

당신: "에이미, 룰렛 돌려봐!"
에이미: "좋아, 항목을 말씀해주세요" → 룰렛 스킬 시작
```

### 스킬이 인식되려면?

에이미가 당신의 말에서 **특정 키워드**를 찾습니다:

| 스킬 | 키워드 |
|------|--------|
| 투표 | "투표", "투도", "투보" |
| 추첨 | "추첨", "뽑기", "시청자참여" |
| 룰렛 | "룰렛", "룰래", "룰렡" |
| 도네이션 투표 | "도네이션투표", "후원투표", "도네투표" |
| 채팅 요약 | "채팅요약" |
| 방송 제어 | "방송제목", "방송카테", "방송태그", "방제" |
| 설정 변경 | "설정변경", "세팅변경", "네설정", "봇설정" |

**팁**: 한 번 스킬이 시작되면, 그 스킬 내에서 계속 대화할 수 있습니다. 에이미가 추가 질문을 할 때 답변하면 됩니다.

## 대화 중 응답 지연 (반응 속도)

### 왜 에이미가 바로 답변하지 않을까?

더 자연스러운 대화를 만들기 위해 **의도적인 지연**이 있습니다.

**기본 설정**
- 최소 지연: 0.5초
- 최대 지연: 1.0초

에이미는 이 범위 내에서 랜덤하게 응답을 지연시킵니다.

**설정으로 조정 가능**
```
설정 경로: conversation.response_delay_min/max
사용 예: "에이미, 응답 속도를 높여줄래?"
범위: 0~30초 (최소), 0~60초 (최대)
```

## 대화 모드 종료 후

대화 모드가 종료되면:

1. 이전 대화 내용은 **메모리에서 정리**됩니다
2. 다시 호명하면 **새로운 대화**가 시작됩니다
3. 이전 대화와 연결되지 않습니다

## 대화 모드가 작동하지 않을 때

| 증상 | 원인 | 해결책 |
|------|------|--------|
| "에이미야"라고 해도 반응 없음 | 호명 감지 비활성화됨 | 설정에서 `namecall.enabled: true` 확인 |
| 5초마다 이름 감지 | 최소 대기 시간 설정 | `namecall.min_response_gap` 확인 |
| STT가 텍스트를 못 변환 | 마이크 문제 또는 STT 오류 | STT 문제 해결 가이드 참조 |
| AI가 답변을 안 함 | Gemini API 오류 | API 키 확인, 네트워크 상태 확인 |

## 설정 커스터마이징

대화 모드 관련 설정:

```yaml
namecall:
  enabled: true                    # 호명 감지 활성화
  trigger_names: []                # 추가 트리거 단어
  scan_interval: 0.5               # 호명 감지 검사 간격 (초)
  min_response_gap: 3              # 연속 호명 감지 최소 간격 (초)
  conversation_timeout: 30         # 기본 타임아웃 (초)

conversation:
  enabled: true                    # 대화 모드 활성화
  max_turns: 10                    # 최대 턴 수
  response_delay_min: 0.5          # 최소 응답 지연 (초)
  response_delay_max: 1.0          # 최대 응답 지연 (초)
  max_response_length: 500         # 한 응답의 최대 길이
```

## 예시: 실제 대화 시나리오

### 시나리오 1: 단순 대화

```
스트리머: "에이미!"
에이미: "네? 뭐가 필요해?" (1턴)

스트리머: "오늘 방송 재밌어?"
에이미: "물론이지! 시청자 분들 덕분이야" (2턴)

스트리머: (3초 침묵)
→ 타임아웃 경계 중...

스트리머: "그런데 뭐 하고 싶은 거 있어?"
에이미: "흠... 투표를 한 번 해볼까?" (3턴, 질문 포함)
→ 타임아웃이 30초에서 60초로 증가
```

### 시나리오 2: 대화 중 스킬 시작

```
스트리머: "에이미야, 투표를 한 번 시작해봐!"
에이미: "좋아! 투표 항목이 뭘로 할까?" (스킬 시작)

스트리머: "치킨, 피자, 햄버거"
에이미: "오케이! 시청자분들 투표 시작합니다!"
→ 투표 스킬로 전환, 대화 모드 종료

(투표 진행 중...)

(투표 끝남)

스트리머: "에이미!" (새로운 대화 모드 시작)
```
