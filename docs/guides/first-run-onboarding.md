# First-Run Onboarding

처음 `/settings`를 열었을 때 큰 카드처럼 먼저 보이는 화면이 바로 **온보딩 대시보드**야.
이 화면은 “모든 설정을 다 끝내는 곳”이 아니라, **방송을 시작하기 전에 꼭 필요한 것만 먼저 챙기는 시작 화면**이라고 생각하면 쉬워.

![Onboarding dashboard](../assets/screenshots/guides/onboarding-dashboard.png)

---

## 온보딩은 왜 있나?

처음 쓰는 사람은 보통 이런 식으로 막혀.

- 어디부터 해야 하는지 모르겠다
- 설정이 너무 많아 보인다
- 지금 당장 필요한 것과 나중에 해도 되는 걸 구분하기 어렵다

온보딩은 그걸 줄여주려고 있어.

즉,
**“지금 바로 방송에 써보려면 최소 뭐가 필요한가?”**를 먼저 보여주는 화면이야.

---

## 여기서 먼저 보는 항목

현재 온보딩에서는 보통 아래 항목을 먼저 챙기게 돼.

1. **Gemini 연결**
2. **스트리머 / 캐릭터 기본 정보**
3. **자동 반응 on/off**
4. **출력 방식**
5. **OBS / 오버레이**
6. **치지직 연결**

이 중에서 특히 중요한 건,
처음부터 어려운 개발자용 입력을 묻는 게 아니라 **방송자 입장에서 바로 필요한 흐름**으로 보여준다는 점이야.

---

## 꼭 다 채워야 하나?

아니.
처음에는 아래 정도만 해도 충분히 시작 가능해.

- Gemini API 키
- 스트리머 이름
- 캐릭터 이름 / 프리셋
- 주요 콘텐츠
- 출력 채널 하나 이상

나머지는 방송하면서 하나씩 만져도 돼.

---

## 치지직 연결은 어떻게 이해하면 쉬운가?

이번 UI에서는 치지직을 **수동 Channel ID 입력 화면**으로 보지 않는 게 중요해.

지금은 보통 이렇게 생각하면 돼.

1. 먼저 **치지직 연동하기** 버튼을 누른다
2. 브라우저에서 로그인 / 권한 허용을 진행한다
3. 연결이 끝나면 **채널 정보는 자동으로 확인**된다

즉,
대부분의 사용자는 먼저 **OAuth 연결**을 하면 되고,
Channel ID를 직접 적는 건 보통 기본 흐름이 아니야.

> 고급 설정에서 Client ID / Client Secret을 따로 바꾸는 건
> 일반적인 첫 설정보다는 운영/배포용 고급 흐름에 가까워.

---

## 온보딩 화면에서 특히 먼저 볼 것

### 1) AI 연결
가장 먼저 봐야 해.
Gemini API 키가 없으면 대부분의 AI 반응이 멈춰.

### 2) 스트리머 / 캐릭터 정보
이건 봇의 성격과 말투 뼈대를 잡는 부분이야.
값이 비어 있으면 돌아가긴 해도 캐릭터 느낌이 약해져.

### 3) 출력 방식
아무리 AI가 잘 돌아도,
**보여주거나 들려주는 채널이 없으면 방송에서는 거의 안 느껴져.**

### 4) 치지직 연결
치지직을 쓸 거라면,
지금은 보통 **연동 버튼으로 먼저 연결해 보는 방식**으로 이해하면 돼.

---

## 온보딩 이후엔 어디로 가면 되나?

온보딩이 끝났다면 보통 이렇게 가면 쉬워.

1. **Quick Start**로 전체 흐름 다시 확인
2. **Detailed Setup**에서 왜 그런 설정을 하는지 이해
3. 필요할 때 **Settings Guide**에서 세부 탭만 찾아보기
