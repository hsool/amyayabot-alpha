# 오버레이 UI 완벽 가이드

오버레이는 OBS 화면에 표시되는 에이미의 아바타, 말풍선, 게임 화면(투표, 추첨, 룰렛 등)을 담당합니다. 이 문서는 설정 방법과 각 오버레이 UI가 어떻게 작동하는지 설명합니다.

## 오버레이 기본 개념

### 오버레이란?

OBS에서 **브라우저 소스**로 추가하는 웹 페이지입니다. 이 페이지가 실시간으로 에이미의 반응을 표시합니다.

### 2가지 오버레이

| 타입 | 용도 | URL | 내용 |
|------|------|-----|------|
| **메인 오버레이** | 기본 표시 | `http://localhost:3000/overlay` | 아바타 + 말풍선 |
| **인터랙티브 오버레이** | 게임 UI | `http://localhost:3000/interactive-overlay` | 투표/추첨/룰렛 |

---

## OBS에 오버레이 추가하기

### 단계 1: 브라우저 소스 추가

**OBS 실행** → **소스** 섹션 → **"+"** 클릭 → **"브라우저"** 선택

### 단계 2: 메인 오버레이 추가 (아바타 + 말풍선)

| 항목 | 값 |
|------|-----|
| 이름 | `AmyayaBot Overlay` |
| URL | `http://localhost:3000/overlay` |
| 너비 | `1920` |
| 높이 | `1080` |
| 프레임 속도 | `60` |

**추가 설정:**
- "로컬 파일이 아닌 URL" 체크 해제
- "OBS 브라우저 하드웨어 가속 사용" 체크

### 단계 3: 인터랙티브 오버레이 추가 (투표/추첨/룰렛)

**새 소스 추가** → **"브라우저"** 선택

| 항목 | 값 |
|------|-----|
| 이름 | `AmyayaBot Interactive` |
| URL | `http://localhost:3000/interactive-overlay` |
| 너비 | `1920` |
| 높이 | `1080` |
| 프레임 속도 | `60` |

### 단계 4: 배치 조정

OBS에서:
1. 메인 오버레이를 하단에 배치 (아바타 위치)
2. 인터랙티브 오버레이를 상단/중앙에 배치 (게임 화면)
3. 둘 다 배경은 투명 (알파값 0)

**팁**: 설정에서 아바타와 말풍선 위치를 조정하면, OBS에서 소스 크기도 맞춰 조정하면 됩니다.

## 메인 오버레이: 아바타와 말풍선

### 아바타 표시 상태

에이미의 아바타는 **상황에 따라 표정과 움직임**이 바뀝니다.

| 상태 | 표정 | 움직임 |
|------|------|--------|
| 침묵 (기본) | 중립 (neutral) | 가만히 있음 |
| 대화 중 | 감정에 맞는 표정 | 약간의 흔들림 |
| 반응 중 | 행복, 놀람 등 | 큰 움직임 |
| 음성 재생 중 | 대화 중과 동일 | 입술 싱크 (립싱크) |

### 표정의 종류

| 표정 | 언제 표시 | 예 |
|------|-----------|-----|
| `neutral` | 기본 상태 | 대기 중 |
| `happy` | 긍정적 응답 | "좋아! 투표 시작할게!" |
| `surprised` | 놀라움 | "오! 당첨되셨어요?" |
| `angry` | 부정적 상황 | "음... 뭔가 이상한데?" |
| `love` | 애정 표현 | "시청자분들 최고예요!" |
| `sad` | 슬픈 상황 | "아, 아무도 없네요" |
| `excited` | 신나는 상황 | "우와! 투표 시작!" |
| `confused` | 헷갈린 상황 | "뭐라고 했어요?" |
| `sleepy` | 피곤한 상황 | "아... 피곤해" |
| `embarrassed` | 부끄러운 상황 | "어... 그럴 리가" |
| `playful` | 장난스러운 상황 | "ㅋㅋ 재밌다!" |
| `smug` | 자신 있는 상황 | "이건 쉽지!" |
| `scared` | 무서워하는 상황 | "어?! 뭐야?!" |
| `touched` | 감동한 상황 | "감사해요..." |
| `bored` | 지루한 상황 | "흠... 뭐 하지?" |

### 말풍선

#### 말풍선 콘텐츠

- **AI 응답**: 대화 중 에이미의 말
- **방송 정보**: 현재 방송 제목, 시청자 수 등 (설정으로 선택)
- **채팅 요약**: 요약 기능이 활성화되었을 때
- **스킬 결과**: "투표 진행 중입니다" 같은 안내

#### 말풍선 위치 설정

설정에서 조정 가능:

```yaml
bubble:
  enabled: true              # 말풍선 표시 여부
  position: "right"          # "top", "bottom", "left", "right"
  font_size: 16              # 글자 크기
  max_width: 400             # 최대 너비
  display_time: 5            # 표시 시간 (초)
```

#### 말풍선 위치 옵션

| 위치 | 배치 | 용도 |
|------|------|------|
| `right` | 아바타 오른쪽 | 기본 설정 |
| `left` | 아바타 왼쪽 | 왼쪽에 공간이 있을 때 |
| `top` | 아바타 위 | 아래 공간이 없을 때 |
| `bottom` | 아바타 아래 | 위 공간이 없을 때 |

### 아바타 이미지 포맷

지원하는 아바타:

| 포맷 | 특징 | 설정 |
|------|------|------|
| **PNG/JPG 이미지** | 정적 이미지 (움직이지 않음) | `avatar.mode: "png"` |
| **Live2D 모델** | 애니메이션 (움직임, 표정) | `avatar.mode: "live2d"` |

#### PNG 이미지로 표정 표현

PNG를 쓸 때는 각 표정마다 다른 이미지가 필요합니다:

```
avatars/png/
├── neutral.png
├── happy.png
├── surprised.png
├── angry.png
└── ... (각 표정별 파일)
```

설정:

```yaml
avatar:
  current_set: "default"     # 이미지 세트 이름
  mode: "png"                # PNG 사용
  viewport:
    width: 240
    height: 280
    fit: "contain"           # 이미지 크기 맞추기
```

#### Live2D 모델

Live2D는 한 개의 3D 모델로 모든 표정을 표현합니다.

설정:

```yaml
avatar:
  mode: "live2d"
  live2d_model: "model_name"           # 모델 폴더명
  live2d_scale: 1.0                    # 크기 (0.5~2.0)
  live2d_offset_x: 0                   # X축 위치 조정
  live2d_offset_y: 0                   # Y축 위치 조정
  live2d_lipsync: true                 # 립싱크 활성화
  live2d_emotion_map: {...}            # 표정 매핑
  live2d_gesture_motions: {...}        # 제스처 설정
```

### 아바타가 표시되지 않을 때

| 증상 | 원인 | 해결책 |
|------|------|--------|
| 아바타 자리가 비어있음 | 이미지 경로 오류 | 파일 경로 확인 |
| 표정이 안 바뀜 | 단일 이미지 사용 | 표정별 이미지 파일 확인 |
| Live2D 로딩 안 됨 | 모델 파일 손상 | 모델 파일 재설치 |
| 말풍선이 이상한 위치 | 설정 값 오류 | bubble.position 확인 |

## 인터랙티브 오버레이: 게임 화면

### 투표 오버레이

#### 투표 진행 중 화면

```
┌─────────────────────┐
│     투표 진행 중     │
├─────────────────────┤
│ [치킨] ████████ 12표 │
│ [피자] ██████ 8표    │
│[햄버거] ████ 5표    │
├─────────────────────┤
│   남은 시간: 1분    │
└─────────────────────┘
```

**표시 정보:**
- 각 항목과 투표 수
- 진행 상황 바 (progress bar)
- 남은 시간 (타이머가 설정되었을 때)
- 총 투표 수 (또는 숨김 처리)

#### 투표 결과 화면

```
┌─────────────────────┐
│     투표 결과!      │
├─────────────────────┤
│ 1위: 치킨  45표     │
│ 2위: 피자  35표     │
│ 3위: 햄버거 20표    │
│  총 투표: 100표     │
└─────────────────────┘
```

**표시 정보:**
- 우승자 강조 표시
- 각 항목별 최종 투표 수
- 총 투표 수

#### 투표 결과 숨김 모드

결과를 공개하지 않는 설정일 때:

```
┌─────────────────────┐
│   투표 진행 중      │
├─────────────────────┤
│  총 투표: 25표      │
│(자세한 결과는 숨김) │
├─────────────────────┤
│   남은 시간: 2분    │
└─────────────────────┘
```

## 오버레이 설정

### 아바타 관련

```yaml
avatar:
  current_set: "default"           # 이미지 세트
  mode: "png"                      # "png" 또는 "live2d"
  animation_strength: 1.0          # 움직임 크기 (0~2)
  lipsync_strength: 1.0            # 립싱크 강도 (0~2)
  motion_strength: 1.0             # 전체 움직임 (0~2)
  viewport:
    width: 240
    height: 280
    fit: "contain"
    position: "center center"
```

### 말풍선 관련

```yaml
bubble:
  enabled: true                    # 표시 여부
  position: "right"                # "top", "bottom", "left", "right"
  font_size: 16                    # 글자 크기
  max_width: 400                   # 최대 너비
```

### 타이밍 관련

```yaml
timing:
  bubble_display_time: 5           # 말풍선 표시 시간 (초)
```

## 오버레이가 작동하지 않을 때

| 증상 | 원인 | 해결책 |
|------|------|--------|
| 오버레이가 검은 화면 | 서버 미실행 | 백엔드 서버 실행 확인 |
| "WS disconnected" 표시 | 웹소켓 연결 실패 | 서버가 18300 포트 사용 확인 |
| 아바타만 안 보임 | 이미지 경로 오류 | 아바타 파일 경로 확인 |
| 투표/추첨 화면이 안 나옴 | 스킬 미실행 | 음성 명령으로 스킬 시작 확인 |
| 말풍선 텍스트 깨짐 | 폰트 문제 | 브라우저 캐시 삭제 후 새로고침 |

## OBS 설정 팁

### 브라우저 소스 새로고침

우클릭 → "상호작용" → "새로고침" (F5)

### 하드웨어 가속 문제

OBS에서 브라우저 하드웨어 가속이 안 되면:
- "OBS 브라우저 하드웨어 가속 사용" 체크 해제
- 약간 느려질 수 있지만 안정성 증가

### 여러 오버레이 표시 순서

레이어 패널에서 순서 조정:
- **맨 뒤**: 게임 캡처
- **중간**: 메인 오버레이 (아바타)
- **맨 앞**: 인터랙티브 오버레이 (게임 UI)

이렇게 하면 아바타 위에 투표/추첨 화면이 표시됩니다.
